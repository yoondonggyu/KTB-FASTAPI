1. 인증 API 테스트 (Auth)

▼ TestSignup - 회원가입 API

9개 테스트

엔드포인트 정보

POST /api/auth/signup

요청 형식:

{ "email": "user@example.com", "password": "Password123!", "password_check": "Password123!", "nickname": "닉네임", "profile_image_url": "https://example.com/image.jpg" // 선택 }
테스트 ID 상태 기대 코드 테스트 설명
TC-AUTH-001 PASS 201
[성공] 정상 회원가입
Given: 유효한 사용자 데이터 (이메일, 비밀번호, 닉네임)
When: POST /api/auth/signup 호출
Then: 201 Created, message="register_success"
TC-AUTH-002 PASS 201 | 422
[성공] 프로필 이미지 없이 회원가입
Given: profile_image_url이 없는 사용자 데이터
When: POST /api/auth/signup 호출
Then: 201 Created (프로필 이미지는 선택사항)
TC-AUTH-003 PASS 400 | 409
[실패] 중복 이메일로 회원가입 시도
Given: 이미 가입된 이메일
When: 같은 이메일로 재가입 시도
Then: 400 또는 409 Conflict, message="email_already_exists"
TC-AUTH-004 PASS 400 | 422
[실패] 잘못된 이메일 형식
Given: 유효하지 않은 이메일 형식 (@ 없음: "invalid-email")
When: POST /api/auth/signup 호출
Then: 400 또는 422 Unprocessable Entity
TC-AUTH-005 PASS 400 | 422
[실패] 비밀번호 확인 불일치
Given: password와 password_check가 다름
When: POST /api/auth/signup 호출
Then: 400 또는 422, message="password_mismatch"
TC-AUTH-006 PASS 422
[실패] 필수 필드 누락
Given: email, password만 있고 nickname, password_check 누락
When: POST /api/auth/signup 호출
Then: 422 Unprocessable Entity
TC-AUTH-007 PASS 400 | 422
[실패] 빈 닉네임
Given: nickname이 빈 문자열 ("")
When: POST /api/auth/signup 호출
Then: 400 또는 422
TC-AUTH-008 PASS 422
[실패] 잘못된 프로필 URL 형식
Given: 유효하지 않은 URL 형식 ("not-a-valid-url")
When: POST /api/auth/signup 호출
Then: 422 Unprocessable Entity
TC-AUTH-009 PASS 400 | 422
[실패] 약한 비밀번호
Given: 규칙을 만족하지 않는 비밀번호 ("weak")
When: POST /api/auth/signup 호출
Then: 400 또는 422
비밀번호 규칙: 8자 이상, 대소문자 포함, 숫자 포함, 특수문자 포함

Then: 400, 401, 또는 422

GET /api/posts

쿼리 파라미터:

  • page: 페이지 번호 (기본값 1, 최소 1)
  • limit: 페이지당 항목 수 (기본값 10, 최소 1, 최대 100)
  • board_type: 게시판 타입 (couple, family 등)

[성공] 게시글 목록 조회
Given: 게시글이 존재하거나 비어있음
When: GET /api/posts 호출
Then: 200 OK, message="get_posts_success", data에 posts 배열 포함
TC-POST-002 PASS 200
[성공] 페이지네이션 적용 목록 조회
Given: page=1, limit=5 파라미터 지정
When: GET /api/posts?page=1&limit=5 호출
Then: 200 OK, 최대 5개 항목 반환
TC-POST-003 PASS 200
[성공] 게시판 타입별 목록 조회
Given: board_type=couple 파라미터 지정
When: GET /api/posts?board_type=couple 호출
Then: 200 OK, 해당 타입 게시글만 반환
TC-POST-004 PASS 422
[실패] 페이지 번호 0
Given: page=0 (유효하지 않음, 1 이상이어야 함)
When: GET /api/posts?page=0 호출
Then: 422 Unprocessable Entity
TC-POST-005 PASS 422
[실패] 음수 페이지 번호
Given: page=-1
When: GET /api/posts?page=-1 호출
Then: 422 Unprocessable Entity
TC-POST-006 PASS 422
[실패] limit 0
Given: limit=0 (최소 1 이상이어야 함)
When: GET /api/posts?limit=0 호출
Then: 422 Unprocessable Entity
TC-POST-007 PASS 422
[실패] limit 최대값 초과
Given: limit=101 (최대 100)
When: GET /api/posts?limit=101 호출
Then: 422 Unprocessable Entity
