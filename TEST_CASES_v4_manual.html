CURL/Postman 기반 QA 테스트 명세

Manual QA v4.0

작성일: 2024-12-06 | Backend Port: 8000 | Model Port: 8001

테스트 환경 및 사전 조건

테스트 환경
  • Backend API Server: http://localhost:8000
  • Model API Server: http://localhost:8001
  • Database: SQLite (개발), PostgreSQL (운영)
  • 인증 방식: JWT Bearer Token
필수 도구
  • CURL (터미널) 또는 Postman (GUI)
  • JSON Formatter (응답 확인용)
테스트 데이터
# 테스트 계정 Email: test@example.com Password: Test1234! # 관리자 계정 Email: admin@example.com Password: Admin1234!

Auth - 인증 API (8개 테스트)

+
TC-MAN-AUTH-001 회원가입 정상 케이스
High
Endpoint
POST /api/auth/signup
사전 조건 동일한 이메일로 가입된 계정이 없어야 함
테스트 단계
  1. 회원가입 API 호출
  2. 응답 상태 코드 확인 (201)
  3. 응답 본문에 사용자 정보 확인
  4. 로그인 테스트로 계정 생성 확인
CURL 명령어
curl -X POST http://localhost:8000/api/auth/signup \ -H "Content-Type: application/json" \ -d '{ "email": "newuser@example.com", "password": "Test1234!", "nickname": "newuser" }'
예상 결과
Status: 201 Created
Response Body:
{ "id": 1, "email": "newuser@example.com", "nickname": "newuser", "created_at": "2024-12-06T..." }
실제 결과
테스트 실행 후 기록
TC-MAN-AUTH-002 회원가입 - 중복 이메일
High
Endpoint
POST /api/auth/signup
사전 조건 test@example.com 으로 이미 가입된 계정이 있어야 함
테스트 단계
  1. 이미 존재하는 이메일로 회원가입 API 호출
  2. 응답 상태 코드 확인 (400)
  3. 에러 메시지 확인
CURL 명령어
curl -X POST http://localhost:8000/api/auth/signup \ -H "Content-Type: application/json" \ -d '{ "email": "test@example.com", "password": "Test1234!", "nickname": "duplicateuser" }'
예상 결과
Status: 400 Bad Request
Response Body:
{ "detail": "Email already registered" }
TC-MAN-AUTH-003 회원가입 - 비밀번호 유효성 검사 실패
High
Endpoint
POST /api/auth/signup
사전 조건 없음
테스트 단계
  1. 유효하지 않은 비밀번호(특수문자 없음)로 회원가입 요청
  2. 응답 상태 코드 확인 (422)
  3. 유효성 검사 에러 메시지 확인
CURL 명령어
curl -X POST http://localhost:8000/api/auth/signup \ -H "Content-Type: application/json" \ -d '{ "email": "weakpass@example.com", "password": "weakpassword", "nickname": "weakuser" }'
예상 결과
Status: 422 Unprocessable Entity
비밀번호 요구사항: 8-20자, 대문자, 소문자, 숫자, 특수문자 포함
TC-MAN-AUTH-004 로그인 정상 케이스
High
Endpoint
POST /api/auth/login
사전 조건 test@example.com / Test1234! 계정이 생성되어 있어야 함
테스트 단계
  1. 올바른 이메일/비밀번호로 로그인 API 호출
  2. 응답 상태 코드 확인 (200)
  3. access_token 필드 존재 확인
  4. 토큰을 저장하여 이후 테스트에 사용
CURL 명령어
curl -X POST http://localhost:8000/api/auth/login \ -H "Content-Type: application/json" \ -d '{ "email": "test@example.com", "password": "Test1234!" }'
예상 결과
Status: 200 OK
Response Body:
{ "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "token_type": "bearer" }
비고
access_token은 이후 인증이 필요한 API 호출 시 Authorization 헤더에 사용됨
Authorization: Bearer {access_token}
